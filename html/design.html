Position Scheme

This website use a mixture of positioning to better fit the content.

Most of the elements use the default static positioning.

Below is a brief explanation of other meaningful positioning used in this website:

fixed positioning
the nav class and the navigation icon is fixed to the top right corner of the browser body. This also takes the elements out of normal flow. (common.css, Line 39)
relative positioning
the navigation menu, a child of nav, is positioned lower (relative to its original place) to ensure it is not covered by the navigation icon. (common.css, Line 52)
absolute positioning
the input and select in the homepage form is positioned with 10% right empty space (relative to its parent, the form) to align the three boxes vertically. This also takes the elements out of normal flow. (homepage.css, Line 98)

CSS3 Features

Many CSS3 features are introduced in this website, including but not limited to:

Round Corner
  • border-radius is used to create round corner for many div elements. (common.css, Line 6, 10)
  • border-radius is also used for menu_label in order page, with only topleft and topright corner rounded. (order.css, Line 36)
Box Shadow
box-shadow is used to decorate the form in the homepage. (homepage.css, Line 82)
Transitions
  • transition is used to create a color and background color change effect when the mouse hovers over an item in the navigation menu. (common.css, Line 75)
  • A similar effect is also applied for the menu_label in order page. (order.css, Line 37)
Animations
  • animation makes the navigation icon flys in with rotation when the page is loaded. (common.css, Line 46)
  • At the same time, the navigation menu fades in. (common.css, Line 62)

Style grouping

All the CSS content is grouped into 5 files in the /css folder.

No inline CSS is used in this website.

The files are:

common.css
the common style found in index.html and order.html. Including the header, footer, navigation and border style.
